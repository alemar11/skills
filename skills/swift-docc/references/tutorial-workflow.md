# Tutorial Workflow

Use this page when a package already has reference docs and now needs DocC
tutorials, tutorial articles, or tutorial resources.

## Fast Path

1. Start with [Building an interactive tutorial](../assets/DocCDocumentation.docc/Reference%20Syntax/Tutorials%20Syntax/building-an-interactive-tutorial.md) for the end-to-end workflow.
2. Use [Tutorial syntax](../assets/DocCDocumentation.docc/Reference%20Syntax/Tutorials%20Syntax/tutorial-syntax.md) when you need the directive hub.
3. For the table of contents and tutorial pages, open [Tutorials](../assets/DocCDocumentation.docc/Reference%20Syntax/Tutorials%20Syntax/Top-Level%20Directives/Tutorials/Tutorials.md) and [Tutorial](../assets/DocCDocumentation.docc/Reference%20Syntax/Tutorials%20Syntax/Top-Level%20Directives/Tutorial/Tutorial.md).
4. For sections, steps, code listings, and assessments, continue in [tutorials.md](tutorials.md) or [directive-map.md](directive-map.md).
5. If the package does not yet have a `.docc` catalog, go back to [add-a-docc-catalog.md](add-a-docc-catalog.md).
