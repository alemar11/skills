# Document A Swift Package

Use this page when the task is "document this Swift package or library API"
rather than a single DocC feature.

## Fast Path

1. Start with [Documenting a Swift framework or package](../assets/DocCDocumentation.docc/documenting-a-swift-framework-or-package.md) for the end-to-end DocC model.
2. Add or improve public symbol comments with [Writing symbol documentation in your source files](../assets/DocCDocumentation.docc/writing-symbol-documentation-in-your-source-files.md).
3. If the package is actor-heavy, async, or lifecycle-driven, tighten the symbol docs with [document-api-lifecycle-and-behavior.md](document-api-lifecycle-and-behavior.md).
4. If the package needs a landing page, articles, or custom topic groups, add a catalog with [Adding supplemental content to a documentation catalog](../assets/DocCDocumentation.docc/adding-supplemental-content-to-a-documentation-catalog.md) and [Adding structure to your documentation pages](../assets/DocCDocumentation.docc/adding-structure-to-your-documentation-pages.md).
5. Add cross-links with [Linking to symbols and other content](../assets/DocCDocumentation.docc/linking-to-symbols-and-other-content.md).
6. If you need snippets, add them with [Adding code snippets to your content](../assets/DocCDocumentation.docc/adding-code-snippets-to-your-content.md).
7. When the docs are ready to preview or host, continue with [preview-and-publish.md](preview-and-publish.md).
